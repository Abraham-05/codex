# Caso de Uso Principal

## 1. Contexto
Describe en qué situación se usaría tu herramienta.

## 2. Actor principal
¿Quién usa el sistema?

## 3. Objetivo del actor
¿Qué quiere lograr el usuario al ejecutar el programa?

## 4. Flujo principal
1. El usuario abre la terminal.
2. Ejecuta el comando principal.
3. Ingresa parámetros o archivos (si aplica).
4. El sistema procesa la información.
5. El sistema muestra el resultado esperado.

## 5. Flujos alternos
Incluye al menos 2 escenarios alternos (por ejemplo: entrada inválida, archivo inexistente).

## 6. Criterios de éxito
Define cómo sabrás que el caso de uso se completó correctamente.
